# 0002 - 첫 구현 단위는 Coupon 생성/조회의 수직 슬라이스로 시작

| 항목 | 내용 |
|------|------|
| 상태 | accepted |
| 날짜 | 2026-03-05 |
| 분류 | architecture |
| 모듈 | coupon |
| 기술 | Spring Boot, JPA |

## 배경 (Context)

동시성 전략 구현 전에, 기본 흐름(요청 → API → JPA → MySQL)을 먼저 검증할지 전략 패턴부터 구현할지 결정이 필요했다.

## 검토한 선택지 (Options Considered)

1. Coupon 생성/조회의 수직 슬라이스(API + JPA) — 기본 흐름이 먼저 검증되어야 이후 문제를 분리해 학습하기 쉽다
2. 전략 패턴부터 구현 — 초기 진입 복잡도가 높아 학습 초반 디버깅 포인트가 과도하게 늘어난다

## 결정 (Decision)

첫 구현 단위는 Coupon 생성/조회의 수직 슬라이스로 시작한다.

## 결과 (Consequences)

- API → JPA → MySQL 기본 흐름이 검증된 상태에서 전략 패턴을 추가하므로, 문제 발생 시 원인 분리가 쉽다
