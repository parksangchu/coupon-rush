# 0003 - 컨트롤러는 입출력만 담당하고 비즈니스 로직은 CouponService로 분리

| 항목 | 내용 |
|------|------|
| 상태 | accepted |
| 날짜 | 2026-03-05 |
| 분류 | architecture |
| 모듈 | coupon |
| 기술 | Spring Boot |

## 배경 (Context)

컨트롤러에서 레포지토리를 직접 호출하는 구조로 시작했으나, 이후 전략 패턴 확장을 고려하면 책임 분리가 필요했다.

## 검토한 선택지 (Options Considered)

1. CouponService로 분리 — HTTP 계층과 도메인 로직의 책임을 분리하면 테스트 대상이 명확해지고 전략 확장 시 변경 영향 범위를 줄일 수 있다
2. 컨트롤러에서 레포지토리 직접 호출 유지 — 초기 구현은 빠르지만 로직이 쌓일수록 컨트롤러가 비대해지고 재사용/테스트가 어려워진다

## 결정 (Decision)

컨트롤러는 입출력만 담당하고, 쿠폰 비즈니스 로직은 CouponService로 분리한다.

## 결과 (Consequences)

- 테스트 대상이 명확 (서비스 단위 테스트 가능)
- 전략 패턴 추가 시 서비스 레이어만 수정하면 됨
